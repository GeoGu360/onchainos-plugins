## §2 接口映射表

### 2a. 操作列表

| # | 操作 | 类型 | 说明 |
|---|------|------|------|
| 1 | `markets` | 链下查询 | 列出所有借贷市场（名称、利率、TVL、可用流动性） |
| 2 | `positions` | 链下查询 | 查询用户在某市场的当前仓位（抵押物、债务、健康度） |
| 3 | `rates` | 链下查询 | 查询特定市场的借贷利率（borrow APY、lend APY） |
| 4 | `deposit-collateral` | 链上写操作 | 向控制器增加抵押物 |
| 5 | `borrow` | 链上写操作（dry-run only per GUARDRAILS） | 创建贷款（首次）或借更多（增量） |
| 6 | `repay` | 链上写操作（dry-run only per GUARDRAILS） | 偿还债务（部分或全部） |

### 2b. 链下查询接口

#### `markets` — 列出市场

**方法：** eth_call batch 到 OneWayLendingFactory

**Factory 地址：** `0xeA6876DDE9e3467564acBeE1Ed5bac88783205E0` (Ethereum)

**步骤：**
1. `market_count()` → 获取市场总数 N
2. 对每个 index i in [0, N): 批量调用以下 Factory 函数：
   - `names(uint256)` → `0x4622ab03` → market 名称（如 "WETH-long"）
   - `controllers(uint256)` → `0xe94b0dd2` → Controller 地址
   - `vaults(uint256)` → `0x8c64ea4a` → Vault 地址（ERC-4626）
   - `collateral_tokens(uint256)` → `0x49b89984` → 抵押物 token 地址
   - `borrowed_tokens(uint256)` → `0x6fe4501f` → 借贷 token 地址（通常是 crvUSD）
3. 对每个 market，从 Vault 读取：
   - `totalAssets()` → `0x01e1d114` → 供给总量（crvUSD）
4. 通过 Monetary Policy 读利率：见 `rates` 操作

**返回字段：**
```json
{
  "markets": [
    {
      "index": 0,
      "name": "wstETH-long",
      "controller": "0x1e0165dbd...",
      "vault": "0x...",
      "collateral_token": "0x7f39c581...",
      "collateral_symbol": "wstETH",
      "borrow_token": "0xf939e0a0...",
      "borrow_symbol": "crvUSD",
      "total_assets_crvusd": "82706.52",
      "n_loans": 5,
      "total_debt": "29430.16"
    }
  ]
}
```

#### `positions` — 查询用户仓位

**方法：** 对指定 Controller 调用 view 函数

**Controller 参数：** 通过 `--market <name_or_index>` 确定使用哪个 Controller

**eth_call 调用：**

| 函数 | 选择器 | 参数 | 返回值 |
|------|--------|------|--------|
| `loan_exists(address)` | `0xa21adb9e` | user addr | bool |
| `debt(address)` | `0x9b6c56ec` | user addr | uint256 (crvUSD, 1e18) |
| `user_state(address)` | `0xec74d0a8` | user addr | (collateral uint256, stablecoin uint256, debt uint256, n1 int256, n2 int256, rate uint256) |
| `user_prices(address)` | `0x2c5089c3` | user addr | (price_high uint256, price_low uint256) — LLAMMA band prices |
| `health(address,bool)` | `0x8908ea82` | user, false | int256 (health factor * 1e18; <0 = liquidatable) |

**Selector verification (cast sig):**
- `loan_exists(address)` → `0xa21adb9e` ✅
- `debt(address)` → `0x9b6c56ec` ✅
- `user_state(address)` → `0xec74d0a8` ✅
- `user_prices(address)` → `0x2c5089c3` ✅
- `health(address,bool)` → `0x8908ea82` ✅

---

#### `rates` — 查询利率

**步骤：**
1. Factory → `monetary_policies(uint256)` → `0x762e7b92` → MonetaryPolicy 地址
2. MonetaryPolicy → `rate(address)` → `0x0ba9d8ca` → 当前 per-second 借贷利率 (uint256, 1e18 scaled)
3. Vault → `lend_apy()` → `0x1eb25c42` → 存款 APY (uint256, 1e16 scaled = basis points * 100)
4. APY 计算：`borrow_apy = ((1 + rate_per_sec / 1e18) ^ (365*24*3600) - 1) * 100`

**Selector verification:**
- `monetary_policies(uint256)` → `0x762e7b92` ✅
- `rate(address)` → `0x0ba9d8ca` ✅
- `lend_apy()` → `0x1eb25c42` ✅
- `borrow_apy()` → `0x3ca97d20` ✅

**Ancillary:**
- `min_rate()` → `0xd22565a8` on MonetaryPolicy
- `max_rate()` → `0x2c4e722e` (wait — this returned empty; use `min_rate`/`max_rate` for range display)

### 2c. 链上写操作

#### `deposit-collateral` — 增加抵押物

**合约：** Controller (per market)
**函数：** `add_collateral(uint256 collateral, address _for)`
**选择器：** `0x24049e57` ✅ (cast sig verified)

**前提：**
- 用户必须已有活跃贷款 (`loan_exists = true`)，否则用 `create_loan` 创建
- ERC-20 approve: 先 approve Controller 花费 collateral token
  - calldata: `0x095ea7b3` + spender (Controller) padded + amount padded
  - token: collateral_token address (e.g., WETH: `0xc02aaa39b223fe8d0a0e5c4f27ead9083c756cc2`)

**calldata 构造：**
```
selector: 0x24049e57
param1: collateral amount (uint256, 32 bytes, token decimals)
param2: _for address (address, 32 bytes, user wallet)
```

**onchainos 命令：**
```bash
# Step 1: approve
onchainos wallet contract-call --chain 1 \
  --to <collateral_token> \
  --input-data 0x095ea7b3<controller_padded><amount_padded>

# Step 2: add_collateral
onchainos wallet contract-call --chain 1 \
  --to <controller> \
  --input-data 0x24049e57<collateral_amount_padded><user_addr_padded>
```

**⚠️ Note on ETH:** WETH market uses WETH (ERC-20), not native ETH. User must hold WETH. The `use_eth` variant in `remove_collateral` is for withdrawing as ETH; deposit always uses ERC-20 approve flow.

---

#### `borrow` — 创建贷款 / 借更多

**合约：** Controller (per market)

**Case A — 新贷款 (loan_exists = false):**
**函数：** `create_loan(uint256 collateral, uint256 debt, uint256 N)`
**选择器：** `0x23cfed03` ✅

Parameters:
- `collateral`: amount of collateral token (in token decimals)
- `debt`: crvUSD amount to borrow (1e18)
- `N`: number of bands (4–50; typical = 10)

**Case B — 增量借款 (loan_exists = true):**
**函数：** `borrow_more(uint256 collateral, uint256 debt)`
**选择器：** `0xdd171e7c` ✅

**Pre-check:** `max_borrowable(collateral, N)` → `0x9a497196` → max crvUSD borrowable

**calldata for create_loan:**
```
0x23cfed03
<collateral uint256 32 bytes>
<debt uint256 32 bytes>
<N uint256 32 bytes>
```

**calldata for borrow_more:**
```
0xdd171e7c
<collateral uint256 32 bytes>
<debt uint256 32 bytes>
```

**onchainos command:**
```bash
onchainos wallet contract-call --chain 1 \
  --to <controller> \
  --input-data 0x23cfed03<collateral_padded><debt_padded><N_padded>
```

**⚠️ GUARDRAILS:** Borrow = dry-run only. Do not execute real borrow in test environment (liquidation risk with limited ETH).

---

#### `repay` — 还款

**合约：** Controller (per market)
**函数：** `repay(uint256 _d_debt, address _for, int256 max_active_band, bool use_eth)`
**选择器：** `0x37671f93` ✅

**Simple form (repay(_d_debt, _for) 2-param):**
**selector:** `0xacb70815` ✅

**Simplest 1-param (repay(_d_debt)):**
**selector:** `0x371fd8e6` ✅ (verified — returns "Loan doesn't exist" for wallet with no loan, not "function not found")

**Pre-step:** ERC-20 approve crvUSD for Controller
- crvUSD: `0xf939E0A03FB07F59A73314E73794Be0E57ac1b4e`
- approve selector: `0x095ea7b3`

**calldata:**
```
0x371fd8e6
<d_debt uint256 32 bytes>  (use wallet crvUSD balance for full repay)
```

**onchainos command:**
```bash
# Step 1: approve crvUSD
onchainos wallet contract-call --chain 1 \
  --to 0xf939E0A03FB07F59A73314E73794Be0E57ac1b4e \
  --input-data 0x095ea7b3<controller_padded><amount_padded>

# Step 2: repay
onchainos wallet contract-call --chain 1 \
  --to <controller> \
  --input-data 0x371fd8e6<d_debt_padded>
```

**⚠️ GUARDRAILS:** Repay = dry-run only (no real borrow exists in test wallet).

## §3 用户场景

### 场景 1：查询 Curve Lending 市场列表
**用户说：** "Show me all Curve Lending markets on Ethereum"

**Agent 动作序列：**
1. (链下) 调用 `curve-lending markets --chain 1`
2. (eth_call) factory.market_count() → 46
3. (eth_call batch) 读取每个市场 name + controller + vault + total_assets
4. (eth_call) 读取 n_loans + total_debt for each controller
5. 输出表格：市场名、抵押物、总供给、活跃贷款数、总债务

---

### 场景 2：查询 WETH-long 市场利率
**用户说：** "What are the borrow and lend rates for the WETH lending market on Curve?"

**Agent 动作序列：**
1. (链下) 调用 `curve-lending rates --market WETH-long --chain 1`
2. (eth_call) factory.monetary_policies(1) → MonetaryPolicy address
3. (eth_call) mp.rate(controller) → per-second rate → compute borrow APY
4. (eth_call) vault.lend_apy() → lending APY
5. (eth_call) vault.totalAssets() → total liquidity
6. 输出：borrow APY, lend APY, total liquidity

---

### 场景 3：查询用户仓位
**用户说：** "Show my Curve Lending positions on Ethereum"

**Agent 动作序列：**
1. (EVM) resolve wallet: `onchainos wallet balance --chain 1 --output json` → address
2. (链下) 调用 `curve-lending positions --chain 1`
3. For each market, (eth_call) controller.loan_exists(wallet)
4. For markets with loan_exists=true:
   - controller.user_state(wallet) → collateral, stablecoin, debt, n1, n2
   - controller.user_prices(wallet) → liquidation band prices
   - controller.health(wallet, false) → health factor
5. 输出：每个市场的抵押物数量、债务金额、健康度、清算区间

---

### 场景 4：Dry-run 存入抵押物
**用户说：** "I want to deposit 0.001 WETH as collateral in the WETH-long Curve Lending market"

**Agent 动作序列：**
1. (EVM) resolve wallet address
2. 查询 loan_exists → False → would need create_loan
3. 查询 max_borrowable(0.001 ETH, 10) → ~1.75 crvUSD
4. Dry-run: 构造 ERC-20 approve calldata (WETH → Controller)
5. Dry-run: 构造 create_loan calldata (0.001 WETH collateral, min debt, 10 bands)
6. **Ask user to confirm** before proceeding with real transaction
7. (L4 on-chain) approve + add_collateral if confirmed

---

### 场景 5：查询最大可借金额
**用户说：** "How much crvUSD can I borrow if I deposit 1 ETH into the WETH lending market?"

**Agent 动作序列：**
1. (eth_call) controller.max_borrowable(1e18, 10) → ~1750 crvUSD
2. (eth_call) rates → current borrow APY
3. 输出：max crvUSD borrowable, borrow APY, health implications
